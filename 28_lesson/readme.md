## Примеры рекурсивных алгоритмов

У каждой правильной рекурсивной функции есть две основные части:

- **Базовый случай, крайний случай** (условие выхода) - это условие, при котором функция перестает вызывать саму себя и возвращает конкретное значение. Без базового случая функция будет вызывать себя бесконечно, что приведет к переполнению стека вызовов и ошибке.
- **Рекурентное соотношение, рекурсивный шаг** - это часть функции, где она вызывает саму себя, но с измененным аргументом, постепенно приближаясь к базовому случаю.



### Фактрориал
`n! = n * (n - 1)!, при n >= 1`    -> рекурентное соотношение  
`0! = 1` -> крайний случай

### Алгоритм Евклида
Это один из самых эффективных алгоритмов для нахождения набольшего общего делителя (НОД).  
`НОД (a, b) = НОД (b, a mod b), если b не равно 0` -> рекурентное соотношение  
`b = 0` -> крайний случай  
НОД (a, 0) = a -> условие, при b = 0

### Возведение в степень
`a^n = a * a^n-1, при n > 0` -> рекурентное соотношение
`a^n = 1, при n = 0` -> крайний случай  
Быстрое возведение в степень если a чётное:
`(a^2)^(n/2)`

### Числа Фибоначчи
`fib(n) = fib(n-1) + fib(n-2)` -> рекурентное соотношение  
`fib(n) = n, если n = 0 и n = 1` -> крайний случай
